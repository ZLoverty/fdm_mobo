# FDM 贝叶斯优化 — 操作说明

一份给使用者的简明上手指南。这套工具帮你**用尽量少的实验次数，找到让多个目标都尽量好的打印参数**
（例如同时兼顾表面质量和拉伸强度）。它会根据你已测的数据，**建议下一个最值得尝试的参数**。

## 打开

浏览器访 `http://192.168.22.55:5002`。
打开就能看到完整界面，无需安装任何东西。

界面分四块：

```
┌────────────────────────────────────────────────┐
│ 实验: [下拉选择] 新建实验 打开配置    下载数据 下载配置 │  ← 顶部
├──────────────────────────┬─────────────────────┤
│ 实验历史(表格)            │ 当前待测 / 填写结果   │  ← 中部
│                          │ 统计                 │
├──────────────────────────┴─────────────────────┤
│ 可视化：参数空间 + 目标空间(Pareto 前沿)         │  ← 底部
└────────────────────────────────────────────────┘
```

## 完整流程

### 1. 新建一个实验

点 **新建实验**，输入名字（如 `PLA-测试`）。系统会复制一份配置模板并弹出配置窗口。

### 2. 设置参数和目标（配置窗口）

- **参数（搜索维度）**：你能调的打印设置。填 `名称` 和 `下限 / 上限`（搜索范围）。
  例：`fan` 0–100、`flow` 0.9–1.1。可点 `+ 添加参数` / ✕ 删除。
- **优化目标**：你要优化的指标，选 `max`（越大越好）或 `min`（越小越好）。
  例：`surface` max、`TS` max。
- **运行设置**：`初始点个数` = 正式优化前先随机试几个点（建议 6–8）；`每轮建议数` 一般填 1。
- 点 **保存**。（随机种子等内部参数无需关心，已隐藏。）

### 3. 生成起始点

点 **初始化**。系统按你的范围生成若干个均匀分布的起始参数点，填进左侧表格（结果列暂时空着）。

> ⚠️ 初始化会清空该实验已有数据，只在刚建好实验时点。

### 4. 按建议做实验，回填结果

右侧「当前待测」会**大字显示下一个要试的参数**。照着这些参数打印 + 测量，然后：

- 在「填写测量结果」里输入每个目标的实测值 → 点 **提交结果**（或在输入框按回车）。
- 提交后自动跳到下一个待测点，重复，直到所有起始点测完。

> 想改某个已填的值？在左侧表格里**双击**那个参数/结果单元格即可修改。

### 5. 获取下一个最优建议

所有点都测完后，**获取 BO 建议** 按钮变可用。点它，系统会拟合模型、算出
下一个最值得尝试的参数，追加到表格末尾（标记为 `bo`）。

### 6. 循环

回到第 4 步：照新建议做实验 → 提交结果 → 再 **获取 BO 建议**。
如此往复，每一轮都在向更优参数逼近。

## 看懂界面

**左侧表格**颜色含义：
- 橙色行 = 待测（还没填结果）
- 蓝色加粗行 = 当前 Pareto 前沿（目前最优的一批折中方案）

**统计**面板：
- 完成 / 待测数量
- **超体积**：衡量整体优化进展的单一数字，越大越好，可用来判断"还要不要继续"。
- 各目标当前最佳值

**可视化**（可缩放、平移、把鼠标悬停在点上看具体数值）：
- 左图「参数空间」：每个点是一次实验的参数；颜色深浅 = 选中目标的好坏（可用上方单选切换看哪个目标）；
  蓝圈 = Pareto 点，橙色 ✕ = 待测点。
- 右图「目标空间」：横纵轴是两个目标；蓝圈连成的就是 Pareto 前沿。

## 导出数据

- **下载数据**：导出该实验全部记录为 `trials.csv`（参数 + 测量值 + 时间）。
- **下载配置**：导出 `config.yaml`。

## 常见问题

**"至少需要 2 个才能运行 BO"**
　已完成的点不足 2 个。先多回填几个结果。

**"还有未测量的点，先提交结果再获取建议"**
　表格里还有橙色待测行。把它们都填上结果再点建议。

**改了参数/目标的维度后报"配置冲突"**
　改动维度会让旧数据失效。请用 **新建实验** 复制配置再改，或对当前实验点 **初始化** 重新开始。

**切换实验**
　顶部下拉直接选；各实验数据相互独立、随时切换、随时回来继续。
